O que é o Projeto Final?

O Projeto Final é a atividade mais importante da disciplina — vale 30% da nota. Vocês vão resolver um problema real usando técnicas de IA, simulando o fluxo de trabalho profissional completo: definir o problema, coletar dados, construir o modelo, fazer deploy e documentar.

💡 Filosofia do projeto

Não queremos um notebook acadêmico que ninguém vai usar. Queremos algo que vocês possam colocar no portfólio, mostrar numa entrevista de emprego, ou até lançar como produto. Pensem grande, executem com qualidade.

Cronograma

Datas e Entregas

O projeto tem 3 marcos ao longo do semestre. Sem segunda chamada — todas as datas são firmes.

28/05
Semana 10

📋 Entrega 1 — Proposta (1 página)

Documento de 1 página contendo: título do projeto, nomes da equipe, descrição do problema, fonte de dados pretendida, abordagem técnica (quais algoritmos/modelos pretendem usar), e resultado esperado. Entregar como PDF no GitHub.

⚠️ Equipes sem proposta entregue não poderão apresentar o projeto final.

18/06
Semana 12

⚡ Entrega 2 — Checkpoint (5 min por equipe)

Apresentação informal do progresso: dados coletados, primeiros experimentos, dificuldades encontradas. Não precisa de slides — pode ser direto do notebook. Objetivo: garantir que ninguém deixou tudo para o final.

O checkpoint não vale nota separada, mas a ausência impacta a nota de processo do projeto.

09/07
Semana 15

🚀 Entrega 3 — Apresentação Final + Código + Documentação

Apresentação de 15 minutos + 5 minutos de perguntas por equipe. Demonstração ao vivo obrigatória (API, app web, demo em notebook). Código no GitHub + relatório técnico.

Entregáveis

O que a equipe deve entregar

📁 Repositório GitHub

  • Código-fonte organizado e documentado
  • Notebooks Jupyter com experimentos
  • README.md completo (instruções de instalação, uso, descrição)
  • requirements.txt ou environment.yml
  • Commits de todos os membros da equipe

📄 Relatório Técnico

  • 3–5 páginas (sem capa/referências no limite)
  • Introdução e motivação do problema
  • Metodologia: dados, modelos, ferramentas
  • Resultados e métricas de avaliação
  • Limitações e trabalhos futuros
  • Referências (artigos, datasets, docs)

🖥️ Demo Funcional

  • O projeto precisa rodar ao vivo
  • Opções: API (FastAPI/Flask), app web (Streamlit/Gradio), notebook interativo
  • Não precisa estar em produção — mas precisa funcionar na apresentação

🎤 Apresentação

  • 15 minutos + 5 min perguntas
  • Slides (Google Slides, PPT ou PDF)
  • Todos os membros devem apresentar
  • Demo ao vivo obrigatória
  • Estejam preparados para perguntas técnicas

Inspiração

Ideias de Projetos

Vocês podem escolher um dos temas abaixo ou propor um tema próprio (sujeito a aprovação). O importante é que seja um problema real onde IA agrega valor.

🔥 ML Clássico + Dados Tabulares

📈 Previsor de demanda

Prever vendas de comércio local usando dados históricos. Scikit-learn, XGBoost. Dados: Kaggle ou dados reais de algum comércio.

🏥 Predição de diabetes/doença cardíaca

Pipeline completo de ML para diagnóstico preditivo. Datasets públicos de saúde (UCI, Kaggle).

🌊 Previsão de cheias em Belém

Usar dados históricos de maré e chuva para prever alagamentos. dados.gov.br + INMET.

💰 Detector de fraude

Classificar transações como fraudulentas. Dados desbalanceados. Técnicas de balanceamento (SMOTE). Kaggle Credit Card Fraud.

🖼️ Deep Learning + Visão Computacional

🌿 Classificador de espécies da Amazônia

Identificar espécies de plantas/animais via foto. Transfer learning (ResNet/EfficientNet). Dataset próprio ou iNaturalist.

🛰️ Detecção de desmatamento por satélite

Segmentação de imagens de satélite para identificar áreas desmatadas. Dados: Sentinel-2, DETER.

🔍 Detector de EPI em canteiro de obra

Detectar se trabalhadores estão usando capacete/colete. YOLO + dataset de segurança.

📝 OCR para documentos históricos

Extrair texto de documentos antigos escaneados. Tesseract + pós-processamento com NLP.

🤖 LLMs + NLP + Agentes

💊 Chatbot de saúde com RAG

Chatbot que responde sobre medicamentos usando bulas como base de conhecimento. LangChain + ChromaDB + API.

⚖️ Agente para análise de contratos

LLM que lê contratos jurídicos e identifica cláusulas de risco. Prompt engineering + RAG.

📰 Detector de fake news

Classificar notícias como verdadeiras/falsas usando NLP e Transformers. Dataset Fake.br ou similar.

👨‍💻 Assistente de programação

LLM fine-tunado em documentação específica (ex: FastAPI, Django). RAG sobre docs técnicas.

🎓 Tutor de IA para a disciplina

Chatbot que tira dúvidas sobre o conteúdo de CC0121 usando o material das aulas como base. Meta-projeto!

🌿 Assistente ambiental para Amazônia

Agente que consulta dados do INPE + legislação ambiental e responde perguntas sobre desmatamento, unidades de conservação, etc.

🤔 Dica para escolher o tema

Escolham algo que vocês se importam pessoalmente. Projeto bom = motivação alta. Se alguém da equipe trabalha com saúde, façam algo de saúde. Se gostam de games, façam IA para jogos. Se se preocupam com meio ambiente, façam algo para a Amazônia. O melhor projeto é aquele que vocês teriam orgulho de mostrar.

Próximo passo

📋 A Proposta — Entrega 28/05

A proposta é um documento de 1 página (PDF) contendo:

Título do projeto

Nome claro e descritivo. Ex: "Classificador de Espécies de Peixes da Amazônia via Transfer Learning".

Equipe

Nomes completos + matrícula de 3–4 membros. Indiquem o líder/responsável.

Problema

2–3 frases: qual problema vocês vão resolver? Por que é importante? Quem se beneficia?

Dados

Onde vão obter os dados? Já têm acesso? Qual o tamanho aproximado? Precisam coletar?

Abordagem técnica

Quais algoritmos/modelos pretendem usar? ML clássico? Deep learning? LLM + RAG? Justifiquem brevemente.

Resultado esperado

O que o projeto vai entregar? Uma API? Um app? Um notebook com análise? Quais métricas de sucesso?

🖥️ Ação imediata

Agora: formem equipes de 3–4 pessoas. Conversem sobre interesses. Até o fim da aula de hoje, cada equipe deve ter pelo menos um tema candidato. Entrega da proposta: PDF no GitHub até 28/05 (próxima aula — Semana 10).

✅ Proposta BOA

  • Problema claro e específico
  • Dados identificados e acessíveis
  • Abordagem técnica justificada
  • Escopo viável para 11 semanas
  • Resultado concreto e demonstrável

❌ Proposta RUIM

  • "Fazer um chatbot de IA" (vago demais)
  • Sem fonte de dados definida
  • "Vamos usar deep learning" sem justificativa
  • Escopo gigante (ex: "curar câncer com IA")
  • Sem resultado mensurável
Dúvidas

Perguntas Frequentes

Posso usar um dataset que eu mesmo coletar?

Sim! Inclusive é valorizado. Dados originais (ex: fotos de plantas da UFPA, dados de comércios locais) são mais interessantes que datasets prontos do Kaggle. Mas tenham um plano B caso a coleta atrase.

Posso usar ChatGPT/Copilot para escrever código?

Sim, como ferramenta. Mas vocês precisam entender e explicar cada parte. Na apresentação, vou fazer perguntas técnicas sobre o código. Se não souberem responder, perde nota.

Posso fazer o projeto sozinho?

Não. Equipes de 3–4 são obrigatórias. Trabalho em equipe é parte da avaliação e da experiência profissional.

O projeto precisa estar "perfeito"?

Não. Valorizo mais um projeto honesto que reconhece limitações do que um que finge que tudo funciona. Mostrem o que deu certo E o que não deu — isso é ciência.

Posso mudar de tema depois da proposta?

Sim, com aprovação do professor. Mas quanto antes melhor — mudanças tardias prejudicam o prazo.

Preciso gastar dinheiro com APIs?

Não necessariamente. Há opções gratuitas: Hugging Face Inference (grátis), Ollama (local), OpenAI tem créditos gratuitos para estudantes, Google Colab é grátis. Se quiserem usar API paga, o custo típico é $5–10 para um projeto inteiro.
