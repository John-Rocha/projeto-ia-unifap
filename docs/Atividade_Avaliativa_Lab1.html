A Atividade do Século XXI

Esta não é uma atividade de decorar sintaxe. No mercado real de IA, ninguém memoriza a assinatura do train_test_split — todos consultam documentação e usam assistentes de IA. O que separa um bom profissional é saber QUAL técnica aplicar e POR QUÊ. Esta atividade avalia exatamente isso.

🤖LLM LIBERADO. Vocês PODEM usar ChatGPT, Claude, Copilot. A regra: vocês decidem a estratégia, o LLM ajuda a implementar — e vocês devem saber explicar cada linha.

✅ O que VOCÊ decide (vale a nota)

  • Qual algoritmo usar em cada situação
  • Qual métrica importa para o problema
  • Como interpretar os resultados
  • Diagnosticar overfitting e propor a cura
  • Justificar TODA decisão técnica

🤖 O que o LLM pode fazer (ferramenta)

  • Lembrar a sintaxe de uma função
  • Escrever o boilerplate do gráfico
  • Sugerir como tratar valores faltantes
  • Ajudar a debugar um erro
  • Acelerar a digitação do que VOCÊ planejou
⚠️ A regra de ouro

Em cada questão, há um campo "Justifique". Se o LLM escreveu o código mas vocês não souberem explicar a escolha, perdem a maior parte dos pontos. Na correção (e na apresentação), vou perguntar: "por que esse modelo e não outro?". Saibam responder.

Questão 1 · 2,0 pontos · ~25 min

🧭 Escolha da Técnica (sem código)

Para cada cenário abaixo, responda numa célula Markdown: (a) qual técnica/algoritmo você usaria, (b) qual métrica de avaliação, (c) uma frase justificando. Aqui o LLM é seu consultor — mas a decisão final e a justificativa são suas.

Cenário A — 0,4 pt

Um hospital quer prever se um paciente tem uma doença rara (afeta 2% dos casos) a partir de 30 exames. Falso negativo (não detectar a doença) é catastrófico.

→ Qual abordagem? Qual métrica? Por quê?

Cenário B — 0,4 pt

Uma rede de farmácias tem dados de 50 mil clientes (idade, frequência, valor gasto) e quer descobrir grupos naturais de comportamento para campanhas direcionadas. Não há rótulos.

→ Qual abordagem? Como decidir o número de grupos?

Cenário C — 0,4 pt

Uma fintech quer prever o valor (em R$) que um cliente vai gastar no próximo mês, baseado no histórico. A saída é um número contínuo.

→ Classificação ou regressão? Qual métrica de erro?

Cenário D — 0,4 pt

Você tem um dataset tabular com 100 mil linhas e 40 colunas para um problema de classificação. Quer a melhor performance possível sem semanas de trabalho.

→ Que família de algoritmos? Por que não rede neural?

Cenário E — 0,4 pt

Um modelo de previsão de crédito atinge 99% de acurácia no treino e 68% no teste.

→ Qual o diagnóstico? Cite 3 formas de resolver.

💡 Dica de uso do LLM aqui

Pergunte ao LLM: "Quais os prós e contras de usar recall vs precisão em detecção de doença rara?" — e então forme sua própria opinião. Copiar a resposta crua sem entender vai aparecer na justificativa rasa.

Questão 2 · 3,0 pontos · ~50 min

🔬 Pipeline Supervisionado num Dataset NOVO

Esqueçam o Titanic — vocês já o conhecem. Agora é um dataset que vocês nunca viram: previsão de diabetes (Pima Indians). É assim no mercado: chega um dado novo, e você aplica o método.

Python · Ponto de partida (o resto é com vocês)
# ═══ ATIVIDADE — Q2: Previsão de Diabetes ═══ import pandas as pd url = "https://raw.githubusercontent.com/jbrownlee/Datasets/master/pima-indians-diabetes.data.csv" colunas = ['Gravidezes', 'Glicose', 'PressaoArterial', 'EspessuraPele', 'Insulina', 'IMC', 'HistoricoFamiliar', 'Idade', 'Diabetes'] df = pd.read_csv(url, names=colunas) print(df.shape) # (768, 9) — alvo: 'Diabetes' (0/1) df.head()

📋 Suas tarefas

2.1 (0,6 pt) — EDA: investiguem os dados. Atenção: há um problema sério escondido — algumas colunas têm valor 0 onde é biologicamente impossível (glicose 0? pressão 0?). Encontrem e tratem. Justifiquem a estratégia.

2.2 (1,0 pt) — Modelagem: treinem no mínimo 3 modelos diferentes com validação cruzada. Lembrem do que aprenderam: alguns precisam de escalonamento, outros não.

2.3 (0,8 pt) — Avaliação: matriz de confusão + precisão/recall do melhor modelo. Pergunta-chave: num teste de diabetes, qual erro é mais grave — falso positivo ou falso negativo? Sua escolha de métrica deve refletir isso.

2.4 (0,6 pt) — Interpretação: qual variável mais prediz diabetes? Faz sentido clínico? Escrevam 2-3 frases conectando o resultado do modelo com o mundo real.

Questão 3 · 2,0 pontos · ~35 min

🛰️ Não Supervisionado: Padrões Ocultos

Dataset de sementes de trigo (Seeds): 7 medidas geométricas de grãos, sem rótulo de variedade. Descubram quantos tipos de trigo existem — só pelos padrões.

Python · Ponto de partida
import pandas as pd url = "https://raw.githubusercontent.com/jbrownlee/Datasets/master/wheat-seeds.csv" colunas = ['Area', 'Perimetro', 'Compacidade', 'Comprimento', 'Largura', 'Assimetria', 'Sulco', 'Variedade'] df = pd.read_csv(url, names=colunas) print(df.shape) # (210, 8) # 7 medidas geométricas + 'Variedade' (ignorem 'Variedade' no clustering!)

📋 Suas tarefas

3.1 (0,5 pt): apliquem o método do cotovelo. Quantos grupos os dados sugerem? Justifiquem com o gráfico.

3.2 (0,7 pt): rodem K-Means. Como têm 7 dimensões, usem PCA para visualizar os grupos em 2D. Lembrem de escalar os dados antes!

3.3 (0,8 pt): caracterizem cada grupo (qual o grão "médio" de cada cluster?) e proponham uma aplicação real: como uma cooperativa agrícola usaria essa segmentação?

Questão 4 · 2,0 pontos · ~30 min

🔍 Diagnóstico: O Caso do Modelo Doente

Esta questão simula o dia a dia: você herda um modelo de um colega e precisa diagnosticar o que há de errado. Voltem ao dataset de diabetes (Q2).

📋 Suas tarefas

4.1 (0,8 pt): treinem uma árvore de decisão e plotem acurácia de treino vs teste para profundidades de 1 a 20 (a "curva do overfitting"). Identifiquem visualmente: onde começa o overfitting?

4.2 (0,6 pt): qual a profundidade ideal? Justifiquem usando validação cruzada, não o olhômetro.

4.3 (0,6 pt) — Conceitual: expliquem com SUAS palavras (não as do LLM) o que é overfitting, usando uma analogia própria. Citem 3 técnicas para combatê-lo.

🎯 Atenção na 4.3

Esta é a questão onde o uso preguiçoso de LLM mais aparece. Respostas genéricas copiadas ("overfitting é quando o modelo se ajusta demais aos dados de treino...") valem metade. Quero a analogia DE VOCÊS — como explicariam para a sua avó?

Questão 5 · 1,0 ponto · ~15 min

🤖 Como Você Usou a IA?

Esta é a questão mais "2026" da atividade. Numa célula Markdown ao final do notebook, documentem criticamente o uso do LLM. Honestidade vale ponto — fingir que não usou vale zero.

📋 Documentem

5.1 (0,4 pt): citem 2 prompts específicos que usaram e para quê. Ex: "Pedi ao Claude para lembrar a sintaxe do ConfusionMatrixDisplay."

5.2 (0,3 pt): o LLM errou ou sugeriu algo ruim em algum momento? Como perceberam? (Se disserem que nunca errou, não estavam prestando atenção 😉)

5.3 (0,3 pt): qual decisão da atividade foi 100% SUA, sem ajuda do LLM? Por que o LLM não poderia ter decidido por vocês?

💡 Por que essa questão existe

No mercado de 2026, todo mundo usa IA. O que te torna valioso não é evitar a IA — é usá-la com julgamento crítico, saber quando confiar e quando questionar. Esta questão treina exatamente essa habilidade que vai definir a carreira de vocês.
