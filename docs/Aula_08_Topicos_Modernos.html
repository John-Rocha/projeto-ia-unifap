🧠 Como um LLM funciona de verdade

Parte A — A ideia central (teoria curta)

Um LLM (Large Language Model) faz uma coisa só: prever a próxima palavra. Só isso. Toda a "inteligência" emerge de fazer isso muito bem, treinando em trilhões de palavras.

💭 A analogia do autocompletar turbinado

Sabe o autocompletar do celular? Um LLM é isso levado ao extremo: em vez de sugerir 3 palavras, ele modela a probabilidade de qualquer próxima palavra, considerando todo o contexto anterior. "O céu é ___" → ele calcula que "azul" é muito provável.

Três conceitos que vocês precisam dominar:

1
Token — o LLM não vê palavras, vê "pedaços" (tokens). "computador" pode virar "comput" + "ador". O modelo prevê token por token.
2
Contexto — a "janela" de texto que o modelo considera. Tudo que você escreve no prompt entra aqui. Quanto maior, mais o modelo "lembra".
3
Temperatura — controla a aleatoriedade. Temperatura 0 = sempre a palavra mais provável (determinístico). Alta = mais criativo e imprevisível.

Parte B — Vendo tokens e probabilidade na prática

🖥️Façam agora (10 min): vamos visualizar como um texto vira tokens e simular a previsão de próxima palavra.
Python · Tokenização na prática
# Instalar a biblioteca de tokenização da OpenAI (a mesma do GPT) # !pip install tiktoken import tiktoken enc = tiktoken.get_encoding("cl100k_base") # encoder do GPT-4 texto = "Inteligência Artificial na Amazônia é o futuro!" tokens = enc.encode(texto) print(f"Texto: {texto}") print(f"Número de tokens: {len(tokens)}") print(f"IDs dos tokens: {tokens}") # Ver cada pedaço que o modelo "enxerga" print("\nComo o modelo fatia o texto:") for t in tokens: print(f" {t} → '{enc.decode([t])}'")
🔌 Se o tiktoken não instalar

Em raras vezes a instalação falha por rede. Plano B, sem instalar nada: tokens = texto.split() — divide por espaços. Não é o tokenizador real do GPT, mas serve para a ideia de "fatiar texto em pedaços". O conceito é o que importa.

📚 RAG: Dando Memória e Fontes à IA

Parte A — O problema que o RAG resolve

LLMs têm dois problemas sérios: (1) não sabem de nada após a data de treino, e (2) às vezes "alucinam" — inventam fatos com confiança. Como resolver sem retreinar o modelo (que custa milhões)?

💡 A sacada do RAG

RAG = Retrieval-Augmented Generation. Em vez de confiar na memória do modelo, você busca os documentos relevantes primeiro e entrega junto com a pergunta. É a diferença entre uma prova de memória e uma prova com consulta.

O fluxo tem 3 passos:

1
Indexar — quebrar seus documentos em pedaços e guardar.
2
Recuperar — dada uma pergunta, achar os pedaços mais parecidos.
3
Gerar — mandar pergunta + pedaços recuperados para o LLM responder com base nas fontes.

Parte B — Construindo um RAG do zero

🖥️Façam agora (20 min): vamos construir o coração de um RAG — a busca por similaridade — sem nenhuma API, só Python puro. Vocês vão entender o mecanismo real.
Python · O coração do RAG: busca por similaridade
# Nossa "base de conhecimento" sobre a Amazônia docs = [ "O açaí é um fruto amazônico rico em antioxidantes e energia.", "A capivara é o maior roedor do mundo e vive perto de rios.", "O boto-cor-de-rosa habita os rios da bacia amazônica.", "Python é uma linguagem popular em ciência de dados.", "O guaraná é um estimulante natural usado em bebidas energéticas.", ] def tokenize(t): return set(t.lower().replace('.','').replace(',','').split()) def similaridade(a, b): # Similaridade de Jaccard: palavras em comum / palavras totais ta, tb = tokenize(a), tokenize(b) if not ta or not tb: return 0 return len(ta & tb) / len(ta | tb) # ─── PASSO 2 do RAG: RECUPERAR ─── pergunta = "Qual fruto amazônico dá energia?" ranking = sorted([(similaridade(pergunta, d), d) for d in docs], reverse=True) print(f"❓ Pergunta: {pergunta}\n") print("📄 Documentos recuperados (por relevância):") for score, doc in ranking[:3]: print(f" [{score:.2f}] {doc}") print(f"\n✅ Melhor fonte: {ranking[0][1]}")
⚠️ Na vida real

Sistemas reais usam embeddings (vetores densos que capturam significado, não só palavras iguais) e bancos vetoriais como FAISS, Chroma ou Pinecone. Mas o princípio é exatamente este: transformar texto em números e medir distância. Vocês acabaram de construir a versão didática.

🤖 Agentes: LLMs que Raciocinam e Agem

Parte A — De "responder" para "fazer"

Um LLM sozinho só produz texto. Um agente usa o LLM como "cérebro" que decide ações: buscar na web, rodar código, consultar um banco. Ele fecha o ciclo entre pensar e agir.

💭 Lembram da Aula 02?

Voltamos aos agentes — mas agora o "cérebro" não é uma regra if/else, é um LLM. O padrão mais famoso se chama ReAct (Reasoning + Acting): o agente alterna entre pensar e agir até resolver a tarefa.

O loop ReAct funciona assim, repetindo até chegar à resposta:

EtapaO que o agente faz
PensamentoRaciocina sobre o que precisa fazer agora
AçãoEscolhe e usa uma ferramenta (calculadora, busca, código)
ObservaçãoLê o resultado da ferramenta
(repete)Pensa de novo com a nova informação, até concluir

Parte B — Um mini-agente ReAct em Python

🖥️Façam agora (20 min): vamos construir um agente que usa ferramentas. Sem API paga — a "decisão" é simulada para vocês verem o mecanismo do loop.
Python · Agente ReAct com ferramentas
# ─── As FERRAMENTAS que o agente pode usar ─── def calculadora(expr): try: return str(eval(expr, {"__builtins__":{}})) except: return "erro no cálculo" def base_dados(consulta): dados = {"alunos": 40, "equipes": 10, "projetos": 10} return str(dados.get(consulta.lower(), "não encontrado")) ferramentas = {"calculadora": calculadora, "base_dados": base_dados} # ─── O LOOP ReAct ─── def agente(objetivo, plano): print(f"🎯 Objetivo: {objetivo}\n") for passo in plano: tipo, conteudo = passo if tipo == "Pensamento": print(f"🧠 {tipo}: {conteudo}") elif tipo == "Ação": nome, arg = conteudo resultado = ferramentas[nome](arg) print(f"⚙️ Ação: {nome}('{arg}')") print(f"👁️ Observação: {resultado}") elif tipo == "Resposta": print(f"✅ {tipo}: {conteudo}") # O "raciocínio" que um LLM geraria: plano = [ ("Pensamento", "Preciso saber quantos alunos há na turma."), ("Ação", ("base_dados", "alunos")), ("Pensamento", "Agora divido por 4 para achar o nº de equipes."), ("Ação", ("calculadora", "40 / 4")), ("Resposta", "A turma forma 10 equipes de 4 alunos."), ] agente("Quantas equipes de 4 a turma forma?", plano)
🌐 O que muda com um LLM real

Num agente de verdade (LangChain, framework de agentes), o LLM gera o plano sozinho — decide qual ferramenta chamar e quando parar. Vocês forneceram o plano à mão para ver o mecanismo. É assim que o Claude, ao usar ferramentas, ou o Copilot, ao editar seu código, funcionam por dentro.

🔬 Desafio (5 min)

Adicionem uma ferramenta nova (por exemplo, buscar_clima(cidade) que retorna um valor fixo). Depois criem um plano que use as três ferramentas em sequência para responder a uma pergunta composta. Percebam: o poder do agente vem de encadear ferramentas.

⚖️ Ética, Viés e Regulação

Parte A — Por que isso é problema de engenharia

Ética em IA não é "filosofia opcional" — é parte do trabalho técnico. Um modelo enviesado pode negar crédito, emprego ou liberdade a pessoas de forma injusta. E vocês, como quem constrói, são responsáveis.

⚠️ Casos reais que aconteceram

Sistema de recrutamento que rejeitava currículos de mulheres (aprendeu com dados históricos enviesados). Modelo de reincidência criminal que penalizava mais réus negros. Reconhecimento facial que erra muito mais em peles escuras. Nenhum foi mal-intencionado — todos refletiram vieses dos dados.

Parte B — Auditando viés na prática

🖥️Façam agora (15 min): vamos medir viés num modelo de decisão de crédito, do jeito que um auditor de IA faz.
Python · Auditoria de fairness
import numpy as np np.random.seed(42) # Simulação: decisões de crédito de um modelo para dois grupos # (1 = crédito aprovado, 0 = negado) grupo_A = np.random.choice([0,1], 100, p=[0.3, 0.7]) grupo_B = np.random.choice([0,1], 100, p=[0.6, 0.4]) taxa_A = grupo_A.mean() taxa_B = grupo_B.mean() print("📊 AUDITORIA DE FAIRNESS — Modelo de Crédito\n") print(f" Grupo A — taxa de aprovação: {taxa_A:.0%}") print(f" Grupo B — taxa de aprovação: {taxa_B:.0%}") print(f"\n ⚠️ Disparidade: {abs(taxa_A - taxa_B):.0%}") # Regra prática (dos "4/5" ou 80%): se a razão entre grupos < 0.8, há sinal de viés razao = min(taxa_A, taxa_B) / max(taxa_A, taxa_B) print(f" Razão de impacto: {razao:.2f}") if razao < 0.8: print(" 🚨 ALERTA: possível discriminação — investigar o modelo e os dados!") else: print(" ✅ Dentro do limite aceitável.")
🔬 Discussão em dupla (5 min)

Achado o viés, o que fazer? Debatam: (1) dá para "consertar" só no código, ou o problema está nos dados? (2) Quem deve decidir o que é "justo" — o programador, a empresa, a lei? Não há resposta fácil — e é por isso que engenheiros de IA precisam pensar nisso.
