1º Horário · Parte A — Teoria (~15 min)

Aprender sem Professor

Na aula passada, todo dado vinha com gabarito: este passageiro sobreviveu, aquele não. Isso é aprendizado supervisionado — alguém "supervisiona" dizendo as respostas. Mas e quando não há resposta? Um shopping tem dados de 10 mil clientes: idade, renda, frequência, gasto médio. Ninguém rotulou "este é cliente premium" ou "este é caçador de promoções". Os grupos existem — mas estão escondidos nos dados.

Aprendizado não supervisionado é a arte de descobrir estrutura sem gabarito. É como chegar numa festa onde você não conhece ninguém: em meia hora, você percebe os grupos — a roda dos que falam de futebol, o canto dos gamers, a turma da dança. Ninguém te deu uma lista; você inferiu pela proximidade e semelhança. Algoritmos de clustering fazem exatamente isso com dados.

K-Means: o algoritmo da festa

O K-Means é elegantemente simples. Você diz quantos grupos quer (o K), e ele faz uma dança em dois passos que se repete até estabilizar:

Chuta K centros aleatórios

Joga K "bandeiras" no espaço dos dados.

Atribui cada ponto à bandeira mais próxima

Cada cliente entra no grupo do centro mais perto dele.

Move cada bandeira para o centro do seu grupo

A bandeira recalcula sua posição como a média dos seus membros.

Repete 2–3 até nada mudar

Quando as bandeiras param de se mover: convergiu. Grupos encontrados!

💡 E como escolher o K?

O método do cotovelo: rode com K=1,2,3...10 e meça a "compactação" dos grupos (inércia). O gráfico cai rápido e depois estabiliza — o ponto da dobra (o "cotovelo") é o K natural dos dados. Vocês vão ver isso ao vivo já já.

# ═══ CC0121 Aula 07 — Clustering de Clientes ═══ import pandas as pd import numpy as np import matplotlib.pyplot as plt import seaborn as sns from sklearn.cluster import KMeans from sklearn.preprocessing import StandardScaler url = "https://raw.githubusercontent.com/SteffiPeTaffy/machineLearningAZ/master/Machine%20Learning%20A-Z%20Template%20Folder/Part%204%20-%20Clustering/Section%2025%20-%20Hierarchical%20Clustering/Mall_Customers.csv" df = pd.read_csv(url) df.columns = ['ID', 'Genero', 'Idade', 'Renda_kUSD', 'Score_Gastos'] print(df.head()) X = df[['Renda_kUSD', 'Score_Gastos']].values # ── Método do cotovelo: qual K? ── inercias = [] for k in range(1, 11): km = KMeans(n_clusters=k, random_state=42, n_init=10) km.fit(X) inercias.append(km.inertia_) plt.figure(figsize=(8,4)) plt.plot(range(1,11), inercias, 'o-', color='#00d4aa', linewidth=2) plt.axvline(5, color='#f59e0b', linestyle='--', label='O cotovelo!') plt.xlabel('K (número de grupos)'); plt.ylabel('Inércia') plt.title('Método do Cotovelo', fontweight='bold'); plt.legend(); plt.grid(alpha=.3) plt.show() # ── K-Means com K=5 ── km = KMeans(n_clusters=5, random_state=42, n_init=10) df['Tribo'] = km.fit_predict(X) plt.figure(figsize=(10,6)) cores = ['#00d4aa','#3b82f6','#ec4899','#f59e0b','#8b5cf6'] for t in range(5): grupo = df[df['Tribo'] == t] plt.scatter(grupo['Renda_kUSD'], grupo['Score_Gastos'], s=60, c=cores[t], label=f'Tribo {t}', alpha=.7) plt.scatter(km.cluster_centers_[:,0], km.cluster_centers_[:,1], s=350, c='white', marker='*', edgecolors='black', linewidths=1.5, label='Centros') plt.xlabel('Renda anual (k US$)'); plt.ylabel('Score de gastos (0-100)') plt.title('🛍️ As 5 Tribos do Shopping', fontweight='bold', fontsize=13) plt.legend(); plt.grid(alpha=.3); plt.show() # ─── Persona Cards: quem é cada tribo? ─── print("🪪 PERSONA CARDS — conheça as tribos:\n") for t in range(5): g = df[df['Tribo'] == t] print(f" Tribo {t} ({len(g)} pessoas)") print(f" Idade média: {g['Idade'].mean():.0f} anos | Renda: ${g['Renda_kUSD'].mean():.0f}k | Gasto: {g['Score_Gastos'].mean():.0f}/100 | {(g['Genero']=='Female').mean()*100:.0f}% mulheres") print(f" Batize esta tribo: ________________\n")

2º Horário · Parte A — Teoria (~10 min)

PCA: A Sombra Perfeita

Dados reais raramente têm 2 dimensões plotáveis. Um cliente de banco tem 50 atributos; um exame genético, milhares. Como enxergar dados de 50 dimensões? Não dá — nosso cérebro trava em 3.

O PCA (Análise de Componentes Principais) resolve com uma ideia linda: projetar os dados em menos dimensões preservando o máximo de informação. Pense numa escultura 3D e sua sombra na parede: a sombra é 2D, mas se você escolher bem o ângulo da luz, ela ainda revela a forma. PCA encontra matematicamente o melhor ângulo de luz — as direções onde os dados mais variam.

💡 Para que serve na prática

(1) Visualizar dados de alta dimensão em 2D/3D. (2) Acelerar modelos reduzindo features redundantes. (3) Remover ruído mantendo só os componentes principais. É pré-processamento padrão em pipelines profissionais.

💻 Enxergando 13 Dimensões + DBSCAN

O dataset Wine tem 13 medidas químicas de 178 vinhos de 3 vinícolas. Treze dimensões — impossível plotar. Vamos usar PCA para "fotografar" esse espaço em 2D:

Python · PCA + comparação K-Means vs DBSCAN
from sklearn.datasets import load_wine from sklearn.decomposition import PCA from sklearn.cluster import DBSCAN # 178 vinhos × 13 medidas químicas wine = load_wine() X_wine = StandardScaler().fit_transform(wine.data) print(f"Dimensões originais: {X_wine.shape}") # (178, 13) # PCA: 13D → 2D pca = PCA(n_components=2) X_2d = pca.fit_transform(X_wine) print(f"Variância preservada: {pca.explained_variance_ratio_.sum()*100:.1f}%") fig, axes = plt.subplots(1, 3, figsize=(17, 5)) # 1. A verdade (rótulos reais — só para conferência!) axes[0].scatter(X_2d[:,0], X_2d[:,1], c=wine.target, cmap='viridis', s=50) axes[0].set_title('Realidade: 3 vinícolas\n(rótulos verdadeiros)', fontweight='bold') # 2. K-Means (sem nunca ver os rótulos!) km3 = KMeans(n_clusters=3, random_state=42, n_init=10).fit(X_2d) axes[1].scatter(X_2d[:,0], X_2d[:,1], c=km3.labels_, cmap='viridis', s=50) axes[1].set_title('K-Means descobriu sozinho', fontweight='bold') # 3. DBSCAN (descobre K sozinho + detecta outliers) db = DBSCAN(eps=0.6, min_samples=5).fit(X_2d) axes[2].scatter(X_2d[:,0], X_2d[:,1], c=db.labels_, cmap='viridis', s=50) n_outliers = (db.labels_ == -1).sum() axes[2].set_title(f'DBSCAN ({n_outliers} outliers em amarelo)', fontweight='bold') plt.tight_layout(); plt.show()

Executem e comparem os 3 painéis. O K-Means recuperou as 3 vinícolas sem nunca ver os rótulos — quanto ele acertou visualmente? Depois brinquem com o eps do DBSCAN (0.4, 0.6, 0.8): o que acontece com o número de grupos e outliers? Quando DBSCAN é melhor que K-Means? (Dica: grupos com formato de lua crescente.)

⚡ Quick Check

Qual a principal vantagem do DBSCAN sobre o K-Means?

✅ DBSCAN agrupa por densidade: acha o número de grupos sozinho, lida com formas não-esféricas e marca outliers (-1). K-Means assume grupos "redondos" e exige K.
❌ As vantagens do DBSCAN: não exige K, encontra formas arbitrárias, detecta outliers. Em troca, é sensível ao parâmetro eps.
3º Horário · Parte A — Teoria (~15 min)

A Sabedoria das Multidões

Em 1906, o estatístico Francis Galton viu 787 pessoas numa feira tentando adivinhar o peso de um boi. Ninguém acertou — mas a média dos palpites errou por menos de 1%, melhor que qualquer especialista presente. Erros individuais em direções diferentes se cancelam; o sinal comum permanece.

Ensembles aplicam isso a ML: muitos modelos medianos, combinados, vencem um modelo genial. Há duas filosofias principais — e a diferença entre elas é uma das perguntas favoritas de entrevista de emprego:

🌳 Bagging → Random Forest

Paralelo e independente. Treina centenas de árvores, cada uma num sorteio diferente dos dados (bootstrap) e das features. Cada árvore vota; maioria vence. Como um júri: cada jurado vê o caso de um ângulo, e o veredito coletivo é mais confiável que qualquer jurado isolado. Reduz variância (overfitting).

🚀 Boosting → XGBoost

Sequencial e corretivo. Treina um modelo fraco, vê onde ele errou, e treina o próximo focado nos erros do anterior. Repete centenas de vezes. Como estudar refazendo só as questões que você errou. Reduz viés (underfitting). XGBoost dominou competições Kaggle por uma década.

💡 Regra de ouro do mercado (2026)

Para dados tabulares (planilhas, bancos de dados), ensembles de árvores (XGBoost, LightGBM, Random Forest) ainda costumam vencer redes neurais — com fração do custo. Deep learning domina imagem, texto e áudio. Saber quando usar cada um é o que separa o júnior do pleno.

Python · O torneio definitivo (Titanic revisitado)
from sklearn.ensemble import RandomForestClassifier, GradientBoostingClassifier from sklearn.tree import DecisionTreeClassifier from sklearn.linear_model import LogisticRegression from sklearn.model_selection import cross_val_score !pip install xgboost -q from xgboost import XGBClassifier # Recarregar Titanic com as features da aula passada url = "https://raw.githubusercontent.com/datasciencedojo/datasets/master/titanic.csv" df = pd.read_csv(url) df['Sexo'] = (df['Sex']=='female').astype(int) df['Age'] = df['Age'].fillna(df['Age'].median()) df['Fare'] = df['Fare'].fillna(df['Fare'].median()) df['Sozinho'] = ((df['SibSp']+df['Parch'])==0).astype(int) X = df[['Pclass','Sexo','Age','Fare','Sozinho']] y = df['Survived'] # Individuais vs Ensembles modelos = { 'Árvore solo': DecisionTreeClassifier(max_depth=4, random_state=42), 'Regressão Logística': LogisticRegression(max_iter=1000), 'Random Forest (300)': RandomForestClassifier(n_estimators=300, random_state=42), 'Gradient Boosting': GradientBoostingClassifier(random_state=42), 'XGBoost': XGBClassifier(random_state=42, eval_metric='logloss'), } print("Validação cruzada 5-fold (a avaliação honesta):\n") medias = {} for nome, m in modelos.items(): scores = cross_val_score(m, X, y, cv=5) medias[nome] = scores.mean() barra = '█' * int(scores.mean()*40 - 28) print(f"{nome:<22s} {barra} {scores.mean()*100:.1f}% ± {scores.std()*100:.1f}") # Quem importa? Feature importance do Random Forest rf = RandomForestClassifier(n_estimators=300, random_state=42).fit(X, y) imp = pd.Series(rf.feature_importances_, index=X.columns).sort_values() plt.figure(figsize=(8,4)) imp.plot.barh(color='#00d4aa') plt.title('O que mais importava para sobreviver? (Random Forest)', fontweight='bold') plt.xlabel('Importância'); plt.tight_layout(); plt.show() # ─── 🔥 Os passageiros mais POLÊMICOS: onde as 300 árvores se dividem ─── probs = rf.predict_proba(X)[:, 1] df['Incerteza'] = np.abs(probs - 0.5) polemicos = df.nsmallest(5, 'Incerteza') print("⚖️ TRIBUNAL DIVIDIDO — as 300 árvores não chegam a acordo:\n") for _, r in polemicos.iterrows(): nome = r['Name'].split(',')[0] sexo = '♂' if r['Sexo']==0 else '♀' real = '🛟 viveu' if r['Survived'] else '💀 morreu' print(f" {nome} ({sexo}, {r['Age']:.0f} anos, {r['Pclass']}ª classe): {probs[r.name]*100:.0f}% de chance → realidade: {real}") print("\n💭 Sunderland e Rush: mesma idade, mesma classe, mesmo sexo. Um viveu, outro morreu.") print(" O modelo não tem como saber — e essa é a lição: previsão tem limites.")

⚡ Quick Check

Qual a diferença entre bagging (Random Forest) e boosting (XGBoost)?

✅ Perfeito! Bagging = júri independente votando (reduz variância). Boosting = corrente de modelos onde cada elo conserta o anterior (reduz viés).
❌ Bagging: paralelo e independente (júri). Boosting: sequencial e corretivo (refazer as questões erradas). Essa é pergunta clássica de entrevista!
