1º Horário · Parte A — Busca Local (~10 min teoria)

Busca Local: Otimização sem Caminho

Na Aula 03, buscávamos um caminho (início → objetivo). Agora o problema é diferente: queremos encontrar o melhor estado — sem nos importar com o caminho. Exemplos: posicionar rainhas, otimizar rota de entrega, treinar pesos de rede neural.

🔍 Busca Sistemática (Aula 03)

Explora o espaço todo. Guarda caminho. BFS, DFS, A*. Garante ótimo. Não escala para espaços enormes.

⛰️ Busca Local (Aula 04)

Mantém só estado atual. Move para vizinho melhor. Rápida, pouca memória. Pode ficar presa em ótimos locais.

Hill Climbing (Subida de Encosta)

Algoritmo mais simples de busca local: olha os vizinhos, move para o melhor. Repete até não ter vizinho melhor. Como subir uma montanha no nevoeiro — só sente o terreno ao redor.

⚠️ Problemas do Hill Climbing

Ótimos locais: topo de morro, mas não da montanha. Platôs: área plana, sem direção. Cristas: progresso em zigue-zague. Solução? Simulated Annealing e Algoritmos Genéticos (2º horário).

1º Horário · Parte B — N-Rainhas com Hill Climbing (~40 min)

💻 O Problema das N-Rainhas

Colocar N rainhas num tabuleiro N×N sem que nenhuma ataque outra (mesma linha, coluna ou diagonal). Para N=8: mais de 4 bilhões de configurações possíveis!

⌨️Criem CC0121_Aula04_BuscaLocal.ipynb e copiem o código.
Python · N-Rainhas com Hill Climbing
# ═══ CC0121 Aula 04 — N-Rainhas com Hill Climbing ═══ import numpy as np import matplotlib.pyplot as plt import random, time def conflitos(estado): """Conta pares de rainhas que se atacam.""" n = len(estado) ataques = 0 for i in range(n): for j in range(i+1, n): if estado[i] == estado[j]: # mesma linha ataques += 1 if abs(estado[i] - estado[j]) == abs(i - j): # diagonal ataques += 1 return ataques def hill_climbing(n=8, max_iter=1000): """Hill Climbing para N-Rainhas. Estado = lista de linhas por coluna.""" estado = [random.randint(0, n-1) for _ in range(n)] custo = conflitos(estado) historico = [custo] for it in range(max_iter): if custo == 0: return estado, historico, "sucesso" # Gerar todos os vizinhos (mover 1 rainha) melhor_viz = estado[:] melhor_custo = custo for col in range(n): for linha in range(n): if estado[col] != linha: viz = estado[:] viz[col] = linha c = conflitos(viz) if c < melhor_custo: melhor_viz = viz melhor_custo = c if melhor_custo >= custo: return estado, historico, "preso" # ótimo local! estado = melhor_viz custo = melhor_custo historico.append(custo) return estado, historico, "max_iter" # ─── Rodar e visualizar ─── estado, hist, status = hill_climbing(n=8) print(f"Status: {status} | Conflitos finais: {conflitos(estado)} | Iterações: {len(hist)}") fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(12, 5)) # Tabuleiro board = np.zeros((8,8)) for i in range(8): for j in range(8): board[i][j] = (i+j) % 2 ax1.imshow(board, cmap='binary', alpha=0.3) for col, linha in enumerate(estado): ax1.text(col, linha, '♛', ha='center', va='center', fontsize=24, color='#00d4aa' if status=='sucesso' else '#ef4444') ax1.set_title(f'8-Rainhas ({status})', fontweight='bold') ax1.set_xticks([]); ax1.set_yticks([]) # Curva de conflitos ax2.plot(hist, 'o-', color='#00d4aa', linewidth=2, markersize=4) ax2.set_title('Conflitos ao longo das iterações', fontweight='bold') ax2.set_xlabel('Iteração'); ax2.set_ylabel('Conflitos'); ax2.grid(alpha=0.3) plt.tight_layout(); plt.show()
🖥️ Façam agora — 10 min

Executem várias vezes. Às vezes dá "sucesso", às vezes "preso" (ótimo local). Contem: em 10 execuções, quantas vezes resolve? Essa taxa de sucesso é a limitação do Hill Climbing.

⚡ Quick Check

Por que o Hill Climbing fica "preso"?

✅ Ótimo local: todos os vizinhos são piores ou iguais, mas existe uma solução melhor longe dali. O algoritmo não consegue "descer para subir".
❌ Hill Climbing fica preso em ótimos locais — não há vizinho melhor, mas a solução global está em outra região do espaço.
2º Horário · Parte A — Simulated Annealing (~10 min teoria + 20 min código)

Simulated Annealing: Aceitar o Pior para Encontrar o Melhor

Inspirado na metalurgia: metal quente aceita mudanças grandes; ao esfriar, aceita cada vez menos. SA começa aceitando movimentos ruins (para escapar de ótimos locais) e gradualmente fica mais exigente.

💡 A ideia-chave

Probabilidade de aceitar vizinho pior: P = e−ΔE/T. Temperatura T começa alta (aceita quase tudo) e diminui (fica seletivo). Quando T→0, comporta-se como Hill Climbing.

Python · Simulated Annealing para N-Rainhas
import math def simulated_annealing(n=8, temp_init=4.0, cooling=0.995, max_iter=10000): estado = [random.randint(0, n-1) for _ in range(n)] custo = conflitos(estado) historico = [custo] temp = temp_init for it in range(max_iter): if custo == 0: return estado, historico, "sucesso" # Vizinho aleatório: mover rainha aleatória viz = estado[:] col = random.randint(0, n-1) viz[col] = random.randint(0, n-1) delta = conflitos(viz) - custo # Aceitar se melhor OU com probabilidade e^(-delta/T) if delta < 0 or random.random() < math.exp(-delta / max(temp, 0.001)): estado = viz custo = conflitos(viz) historico.append(custo) temp *= cooling # esfriar! return estado, historico, "sucesso" if custo == 0 else "max_iter" # Comparar taxa de sucesso: HC vs SA n_testes = 100 hc_ok = sum(1 for _ in range(n_testes) if hill_climbing(8)[2] == "sucesso") sa_ok = sum(1 for _ in range(n_testes) if simulated_annealing(8)[2] == "sucesso") print(f"Taxa de sucesso (100 testes):") print(f" Hill Climbing: {hc_ok}%") print(f" Simulated Annealing: {sa_ok}%")

2º Horário · Parte B — Algoritmo Genético (~25 min)

💻 Algoritmo Genético para N-Rainhas

Inspirado na evolução: população de soluções que evolui por seleção, crossover e mutação.

Python · Algoritmo Genético
def algoritmo_genetico(n=8, pop_size=100, geracoes=500, mutacao=0.1): # População inicial aleatória pop = [[random.randint(0,n-1) for _ in range(n)] for _ in range(pop_size)] historico = [] for gen in range(geracoes): # Fitness = inverso dos conflitos fitness = [1 / (1 + conflitos(ind)) for ind in pop] melhor = pop[np.argmax(fitness)] historico.append(conflitos(melhor)) if conflitos(melhor) == 0: return melhor, historico, "sucesso" # Seleção por torneio def torneio(): a, b = random.sample(range(pop_size), 2) return pop[a] if fitness[a] > fitness[b] else pop[b] nova_pop = [melhor[:]] # elitismo while len(nova_pop) < pop_size: pai, mae = torneio(), torneio() # Crossover de 1 ponto corte = random.randint(1, n-1) filho = pai[:corte] + mae[corte:] # Mutação if random.random() < mutacao: filho[random.randint(0,n-1)] = random.randint(0,n-1) nova_pop.append(filho) pop = nova_pop melhor = min(pop, key=conflitos) return melhor, historico, "sucesso" if conflitos(melhor)==0 else "max_gen" # Comparar todos ag_ok = sum(1 for _ in range(n_testes) if algoritmo_genetico(8)[2]=="sucesso") print(f"\nTaxa de sucesso (100 testes, 8-Rainhas):") print(f" Hill Climbing: {hc_ok}%") print(f" Simulated Annealing: {sa_ok}%") print(f" Algoritmo Genético: {ag_ok}%") nomes = ['Hill Climbing', 'Simulated Annealing', 'Alg. Genético'] taxas = [hc_ok, sa_ok, ag_ok] cores = ['#ef4444', '#f59e0b', '#00d4aa'] plt.figure(figsize=(8,4)) bars = plt.bar(nomes, taxas, color=cores, edgecolor='white', linewidth=0.5) for b, v in zip(bars, taxas): plt.text(b.get_x()+b.get_width()/2, b.get_height()+1, f'{v}%', ha='center', fontweight='bold') plt.ylabel('Taxa de sucesso (%)'); plt.title('Comparação: Busca Local para 8-Rainhas', fontweight='bold') plt.ylim(0,110); plt.grid(axis='y', alpha=0.3); plt.tight_layout(); plt.show()
🖥️ Façam agora — 10 min

Executem e vejam o gráfico comparativo. Qual algoritmo ganha? Testem com n=20 (20-Rainhas) — os resultados mudam? Experimentem aumentar pop_size e mutacao.

⚡ Quick Check

No Algoritmo Genético, qual operador introduz diversidade impedindo convergência prematura?

✅ A mutação introduz variação aleatória, impedindo que a população inteira convirja para o mesmo ótimo local.
❌ A mutação é o operador de diversidade. Crossover combina soluções existentes; seleção escolhe as melhores; elitismo preserva a melhor.
3º Horário · Parte A — Busca Adversarial (~15 min teoria)

Jogos e Minimax

Agora o ambiente tem um oponente. Não basta otimizar — precisa considerar que o adversário vai tentar te atrapalhar. É o domínio da busca adversarial.

💡 Minimax em 30 segundos

MAX (você) quer maximizar o score. MIN (oponente) quer minimizar. Minimax explora toda a árvore do jogo, assumindo que o oponente joga perfeitamente. Garante a melhor jogada possível contra um adversário ótimo.

ConceitoDescrição
Estado terminalFim do jogo: vitória, derrota ou empate
UtilidadeScore do estado terminal: +1 (vitória), -1 (derrota), 0 (empate)
Nó MAXEscolhe a ação com MAIOR valor (seu turno)
Nó MINEscolhe a ação com MENOR valor (turno do oponente)
3º Horário · Parte B — Jogo da Velha com Minimax (~35 min)

💻 Jogo da Velha Imbatível

Python · Jogo da Velha com Minimax
# ═══ Jogo da Velha com Minimax ═══ def vencedor(board): linhas = [board[i*3:(i+1)*3] for i in range(3)] cols = [[board[i+j*3] for j in range(3)] for i in range(3)] diags = [[board[0],board[4],board[8]], [board[2],board[4],board[6]]] for trio in linhas + cols + diags: if trio == ['X']*3: return 'X' if trio == ['O']*3: return 'O' return None def terminal(board): return vencedor(board) is not None or ' ' not in board def minimax(board, is_max): v = vencedor(board) if v == 'X': return 1, None if v == 'O': return -1, None if terminal(board): return 0, None best_move = None if is_max: best = -999 for i in range(9): if board[i] == ' ': board[i] = 'X' val, _ = minimax(board, False) board[i] = ' ' if val > best: best, best_move = val, i else: best = 999 for i in range(9): if board[i] == ' ': board[i] = 'O' val, _ = minimax(board, True) board[i] = ' ' if val < best: best, best_move = val, i return best, best_move def mostrar(board): for i in range(3): print(' | '.join(board[i*3:(i+1)*3])) if i < 2: print('---------') # Jogar contra a IA! board = [' ']*9 print("Você é O, IA é X. Posições 0-8:") print("0|1|2\n3|4|5\n6|7|8\n") # IA começa _, move = minimax(board, True) board[move] = 'X' print(f"IA jogou na posição {move}:") mostrar(board) # Loop do jogo (interativo no Colab) while not terminal(board): pos = int(input("Sua jogada (0-8): ")) if board[pos] != ' ': print("Posição ocupada!"); continue board[pos] = 'O' if not terminal(board): _, move = minimax(board, True) board[move] = 'X' print(f"IA jogou na posição {move}:") mostrar(board) v = vencedor(board) print(f"\n{'IA venceu!' if v=='X' else 'Você venceu!' if v=='O' else 'Empate!'}")
🖥️ Façam agora — 15 min

Joguem contra a IA! Tentem vencer. Spoiler: é impossível. Minimax garante que a IA nunca perde no jogo da velha. Discutam em dupla: por que não dá para ganhar?

⚡ Quick Check

No Minimax, o que o jogador MIN faz?

✅ MIN escolhe a jogada que minimiza o score — assume adversário perfeito. Isso garante a melhor resposta possível.
❌ MIN minimiza o score. MAX maximiza. Juntos, exploram toda a árvore assumindo jogo perfeito de ambos.
4º Horário · Parte A — Poda Alfa-Beta (~10 min teoria + 20 min código)

Poda Alfa-Beta: Minimax Inteligente

Minimax explora TODA a árvore. Alfa-Beta corta ramos que não podem influenciar a decisão final — mesmo resultado, muito menos nós.

💡 A ideia

α = melhor valor que MAX pode garantir. β = melhor valor que MIN pode garantir. Se α ≥ β, pode podar (cortar) — esse ramo nunca será escolhido. No melhor caso, reduz de O(bd) para O(bd/2) — raiz quadrada!

Python · Minimax com Poda Alfa-Beta
nos_minimax = 0 nos_alfabeta = 0 def minimax_count(board, is_max): global nos_minimax; nos_minimax += 1 v = vencedor(board) if v == 'X': return 1 if v == 'O': return -1 if terminal(board): return 0 vals = [] for i in range(9): if board[i] == ' ': board[i] = 'X' if is_max else 'O' vals.append(minimax_count(board, not is_max)) board[i] = ' ' return max(vals) if is_max else min(vals) def alfabeta(board, is_max, alpha=-999, beta=999): global nos_alfabeta; nos_alfabeta += 1 v = vencedor(board) if v == 'X': return 1 if v == 'O': return -1 if terminal(board): return 0 if is_max: val = -999 for i in range(9): if board[i] == ' ': board[i] = 'X' val = max(val, alfabeta(board, False, alpha, beta)) board[i] = ' ' alpha = max(alpha, val) if alpha >= beta: break # ← PODA! return val else: val = 999 for i in range(9): if board[i] == ' ': board[i] = 'O' val = min(val, alfabeta(board, True, alpha, beta)) board[i] = ' ' beta = min(beta, val) if alpha >= beta: break # ← PODA! return val # Comparar nós explorados board_vazio = [' ']*9 nos_minimax = 0; minimax_count(board_vazio[:], True) nos_alfabeta = 0; alfabeta(board_vazio[:], True) print(f"Minimax puro: {nos_minimax:>8,} nós explorados") print(f"Alfa-Beta: {nos_alfabeta:>8,} nós explorados") print(f"Redução: {nos_minimax/nos_alfabeta:.1f}x menos nós! 🚀")
🖥️ Façam agora — 10 min

Executem e vejam a diferença. Alfa-Beta deve explorar ~5-10x menos nós que Minimax puro — mesmo resultado, muito mais rápido. Em jogos maiores (xadrez) essa otimização é essencial.
