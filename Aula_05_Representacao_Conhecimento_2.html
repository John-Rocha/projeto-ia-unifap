1º Horário · Parte A — Lógica Proposicional (~15 min teoria)

Representando Conhecimento com Lógica

Antes do deep learning, a IA era dominada pela representação simbólica: conhecimento representado por fórmulas lógicas que a máquina pode manipular formalmente. Isso ainda é fundamental para sistemas que precisam ser explicáveis e auditáveis.

Lógica proposicional — o básico

💡 Implicação material

P → Q é falso SOMENTE quando P é verdadeiro e Q é falso. "Se chove, levo guarda-chuva" — só é mentira se chove e eu não levo. Se não chove, a promessa não é quebrada.

1º Horário · Parte B — Motor de inferência em Python (~35 min)

💻 Construindo um Motor de Inferência

Vamos criar um sistema que recebe fatos e regras em lógica proposicional e infere novos fatos automaticamente.

⌨️Criem CC0121_Aula05_Conhecimento.ipynb.
Python · Motor de Inferência Proposicional
# ═══ CC0121 Aula 05 — Motor de Inferência ═══ class BaseConhecimento: """Base de conhecimento com fatos e regras → inferência por encadeamento para frente.""" def __init__(self): self.fatos = set() self.regras = [] # lista de (premissas, conclusão) def adicionar_fato(self, fato): self.fatos.add(fato) print(f" ✅ Fato: {fato}") def adicionar_regra(self, premissas, conclusao): """premissas = lista de strings, conclusao = string""" self.regras.append((set(premissas), conclusao)) print(f" 📐 Regra: {' ∧ '.join(premissas)} → {conclusao}") def inferir(self): """Encadeamento para frente: aplica regras até não inferir mais nada.""" print(f"\n🔍 Iniciando inferência com {len(self.fatos)} fatos...") novos = True rodada = 0 while novos: novos = False rodada += 1 for premissas, conclusao in self.regras: if premissas.issubset(self.fatos) and conclusao not in self.fatos: self.fatos.add(conclusao) novos = True print(f" 💡 Rodada {rodada}: {' ∧ '.join(premissas)} → {conclusao}") print(f"\n📋 Fatos finais: {sorted(self.fatos)}") def consultar(self, pergunta): resp = pergunta in self.fatos print(f" ❓ {pergunta}? → {'✅ SIM' if resp else '❌ NÃO'}") return resp # ─── Exemplo: Clima e atividades ─── print("═══ Base de Conhecimento: Clima ═══") kb = BaseConhecimento() # Fatos kb.adicionar_fato("chovendo") kb.adicionar_fato("frio") # Regras kb.adicionar_regra(["chovendo"], "levar_guarda_chuva") kb.adicionar_regra(["frio"], "levar_casaco") kb.adicionar_regra(["chovendo", "frio"], "risco_gripe") kb.adicionar_regra(["risco_gripe"], "tomar_vitamina_c") kb.adicionar_regra(["levar_guarda_chuva", "levar_casaco"], "preparado_para_sair") # Inferir! kb.inferir() # Consultas print() kb.consultar("preparado_para_sair") kb.consultar("tomar_vitamina_c") kb.consultar("dia_ensolarado")
🖥️ Façam agora — 10 min

Executem e observem a cadeia de inferências: fatos simples ("chovendo", "frio") levam a conclusões em cascata. Adicionem novos fatos (ex: "vento_forte") e novas regras. O sistema infere automaticamente!

⚡ Quick Check

No encadeamento para frente (forward chaining), o sistema:

✅ Forward chaining: fatos → regras → novos fatos → mais regras... até estabilizar. A opção B seria backward chaining (Prolog).
❌ Forward chaining parte dos fatos e vai aplicando regras até não inferir mais nada. Backward chaining parte do objetivo.
2º Horário · Sistema Especialista (~50 min)

💻 Sistema Especialista de Diagnóstico

Sistemas especialistas foram a primeira "IA aplicada" com sucesso comercial (anos 80). Usam regras IF-THEN capturadas de especialistas humanos. Vamos construir um para diagnóstico médico simplificado.

Python · Sistema Especialista Interativo
# ═══ Sistema Especialista — Diagnóstico Simplificado ═══ class SistemaEspecialista: def __init__(self, nome): self.nome = nome self.fatos = set() self.regras = [] self.diagnosticos = [] def regra(self, sintomas, diagnostico, confianca=1.0): self.regras.append({ 'sintomas': set(sintomas), 'diagnostico': diagnostico, 'confianca': confianca }) def perguntar(self, sintoma): resp = input(f" Você tem {sintoma}? (s/n): ").strip().lower() if resp == 's': self.fatos.add(sintoma) return True return False def diagnosticar(self): print(f"\n🏥 {self.nome}") print("Responda as perguntas sobre seus sintomas:\n") # Coletar todos os sintomas únicos todos_sintomas = set() for r in self.regras: todos_sintomas.update(r['sintomas']) for sintoma in sorted(todos_sintomas): self.perguntar(sintoma) # Verificar regras print(f"\n🔍 Analisando {len(self.fatos)} sintomas contra {len(self.regras)} regras...\n") resultados = [] for r in self.regras: match = len(r['sintomas'] & self.fatos) total = len(r['sintomas']) if match > 0: score = (match / total) * r['confianca'] resultados.append((r['diagnostico'], score, match, total)) resultados.sort(key=lambda x: -x[1]) if resultados: print("📋 Possíveis diagnósticos:") for diag, score, match, total in resultados: bar = '█' * int(score * 20) + '░' * (20 - int(score * 20)) print(f" {bar} {score*100:5.1f}% {diag} ({match}/{total} sintomas)") else: print(" Nenhum diagnóstico encontrado com os sintomas informados.") # ─── Criar o sistema ─── doc = SistemaEspecialista("Sistema de Diagnóstico CC0121") doc.regra(["febre", "tosse", "dor_corpo", "fadiga"], "Gripe", 0.9) doc.regra(["febre", "tosse", "perda_olfato"], "COVID-19 (suspeita)", 0.85) doc.regra(["espirros", "coriza", "coceira_olhos"], "Alergia", 0.95) doc.regra(["febre", "dor_cabeca", "dor_corpo", "manchas_pele"], "Dengue (suspeita)", 0.8) doc.regra(["dor_garganta", "febre", "dor_engolir"], "Amigdalite", 0.85) doc.regra(["tosse", "coriza", "espirros"], "Resfriado", 0.9) doc.diagnosticar()
🖥️ Façam agora — 15 min

Executem e respondam as perguntas. O sistema dá diagnóstico com percentual de confiança! Tarefa: adicionem 3 novas doenças com sintomas próprios. Testem cenários onde mais de um diagnóstico aparece.

🤔 Discussão

Sistemas especialistas dominaram a IA nos anos 80. Por que caíram em desuso? Problemas: difícil adquirir regras de especialistas, frágeis (não lidam com incerteza), não aprendem. Hoje voltam combinados com ML (sistemas neuro-simbólicos).

⚡ Quick Check

Qual foi a principal limitação dos sistemas especialistas dos anos 80?

✅ Sistemas especialistas eram difíceis de construir (aquisição de conhecimento), frágeis fora do domínio, e não aprendiam. Isso levou ao 2º inverno da IA.
❌ A principal limitação era serem frágeis e não aprenderem. Capturar regras de especialistas era caro e as regras não cobriam todos os casos.
3º Horário · Parte A — Grafos de Conhecimento (~15 min teoria)

Grafos de Conhecimento

Grafos de conhecimento representam informação como triplas (sujeito, predicado, objeto). É assim que Google, Wikipedia e Amazon organizam bilhões de fatos.

📊 Estrutura: Triplas SPO

  • (Python, é_um, Linguagem_Programação)
  • (Python, criado_por, Guido_van_Rossum)
  • (PyTorch, usa, Python)
  • (Deep_Learning, requer, PyTorch)

🌐 Exemplos reais

  • Google Knowledge Graph: 500B+ fatos
  • Wikidata: 100M+ itens, aberto
  • DBpedia: extrai de Wikipedia
  • Neo4j, Amazon Neptune: bancos de grafos
3º Horário · Parte B — Construir e consultar KG (~35 min)

💻 Knowledge Graph em Python

Python · Grafo de Conhecimento com NetworkX
# ═══ Grafo de Conhecimento — Ecossistema de IA ═══ import networkx as nx import matplotlib.pyplot as plt # Criar grafo direcionado G = nx.DiGraph() # Adicionar triplas (sujeito, objeto, predicado) triplas = [ ("Python", "Linguagem", "é_um"), ("PyTorch", "Python", "usa"), ("Scikit-learn", "Python", "usa"), ("Transformers", "PyTorch", "usa"), ("GPT", "Transformers", "baseado_em"), ("BERT", "Transformers", "baseado_em"), ("Claude", "Transformers", "baseado_em"), ("GPT", "OpenAI", "criado_por"), ("Claude", "Anthropic", "criado_por"), ("BERT", "Google", "criado_por"), ("LangChain", "Python", "usa"), ("LangChain", "GPT", "integra"), ("LangChain", "Claude", "integra"), ("RAG", "LangChain", "implementado_com"), ("Agente_IA", "LangChain", "implementado_com"), ("AlphaFold", "Deep_Learning", "usa"), ("Deep_Learning", "PyTorch", "implementado_com"), ("CNN", "Deep_Learning", "tipo_de"), ("Transformers", "Deep_Learning", "tipo_de"), ] for s, o, p in triplas: G.add_edge(s, o, relation=p) # Visualizar plt.figure(figsize=(14, 10)) pos = nx.spring_layout(G, k=2, seed=42) # Cores por tipo cores_no = [] for node in G.nodes(): if node in ['OpenAI','Anthropic','Google']: cores_no.append('#f59e0b') elif node in ['GPT','BERT','Claude']: cores_no.append('#3b82f6') elif node in ['Python','PyTorch','Scikit-learn','LangChain']: cores_no.append('#00d4aa') else: cores_no.append('#8b5cf6') nx.draw_networkx_nodes(G, pos, node_color=cores_no, node_size=1500, alpha=0.9) nx.draw_networkx_labels(G, pos, font_size=7, font_weight='bold') nx.draw_networkx_edges(G, pos, edge_color='#555', arrows=True, arrowsize=15, alpha=0.6) edge_labels = nx.get_edge_attributes(G, 'relation') nx.draw_networkx_edge_labels(G, pos, edge_labels, font_size=6, font_color='#888') plt.title('Knowledge Graph: Ecossistema de IA', fontsize=14, fontweight='bold') plt.axis('off'); plt.tight_layout(); plt.show() # ─── Consultas no grafo ─── print("🔍 Consultas ao Knowledge Graph:\n") print("O que usa Python?") for s, o, d in G.edges(data=True): if o == "Python" and d['relation'] == "usa": print(f" → {s}") print("\nQuem criou cada modelo?") for s, o, d in G.edges(data=True): if d['relation'] == "criado_por": print(f" → {s} foi criado por {o}") print(f"\nCaminho de RAG até Python:") path = nx.shortest_path(G, "RAG", "Python") print(f" → {' → '.join(path)}")

⚡ Quick Check

Em um grafo de conhecimento, a informação é armazenada como:

✅ Knowledge graphs usam triplas SPO: (sujeito, predicado, objeto). Ex: (Python, é_um, Linguagem). Cada aresta é uma relação.
❌ A estrutura fundamental é a tripla (sujeito, predicado, objeto). Tabelas = bancos relacionais. Vetores = embeddings de ML.
4º Horário · Parte A — IA Neuro-Simbólica (~10 min teoria + 20 min prática)

Combinando Simbólico + Neural

Em 2026, a fronteira da IA é neuro-simbólica: combinar a flexibilidade dos LLMs com a precisão da lógica formal. RAG (Retrieval-Augmented Generation) é exatamente isso — LLM + base de conhecimento.

🧠 Neural (LLMs)

✅ Flexível, generaliza, linguagem natural
❌ Alucina, caixa-preta, não auditável

📐 Simbólico (Lógica/KG)

✅ Preciso, explicável, auditável
❌ Frágil, não generaliza, difícil escalar

🔗 Neuro-Simbólico = Melhor dos dois mundos

LLM para entender linguagem + Knowledge Graph para fatos verificáveis. O LLM raciocina, o KG garante que os fatos estão corretos. É a base de RAG, agentes com ferramentas, e fact-checking automático.

💻 LLM como motor de raciocínio sobre base de conhecimento

Vamos simular o padrão: o "LLM" recebe uma pergunta + fatos da base e produz uma resposta fundamentada.

Python · Simulando RAG com nossa base de conhecimento
# ═══ Simulação: base de conhecimento + raciocínio ═══ def consultar_kg(grafo, pergunta_tokens): """Busca fatos relevantes no KG baseado em palavras-chave.""" fatos = [] for s, o, d in grafo.edges(data=True): for token in pergunta_tokens: if token.lower() in s.lower() or token.lower() in o.lower(): fatos.append(f"{s} {d['relation']} {o}") return list(set(fatos)) def gerar_prompt_rag(pergunta, fatos): """Gera prompt no padrão RAG: contexto + pergunta.""" contexto = "\n".join(f"- {f}" for f in fatos) prompt = f"""Com base nos seguintes fatos verificados: {contexto} Pergunta: {pergunta} Responda usando APENAS os fatos acima. Se não souber, diga "Não encontrado na base".""" return prompt # Exemplo pergunta = "Quem criou o Claude e qual tecnologia ele usa?" tokens = ["Claude", "criou", "tecnologia"] fatos = consultar_kg(G, tokens) print("📋 Fatos recuperados do Knowledge Graph:") for f in fatos: print(f" → {f}") prompt = gerar_prompt_rag(pergunta, fatos) print(f"\n📝 Prompt RAG gerado:\n{prompt}") print("\n💡 Cole este prompt num LLM (ChatGPT/Claude) e veja a resposta fundamentada!")
🖥️ Façam agora — 10 min

Executem, copiem o prompt gerado e colem num LLM. Comparem: o LLM responde com base nos fatos (sem alucinações) vs responder a mesma pergunta sem o contexto. Essa é a essência do RAG!
